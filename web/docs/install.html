Verity Core Banking Platform

Installation Manual for Implementation Engineers

Version 0.1.0 — May 2026

Prerequisites

  • A Linux server running a recent kernel (5.15+). Bare‑metal is required for production; a VM may be used for evaluation with reduced security guarantees.
  • For production: Intel Xeon with TDX support, or AMD EPYC with SEV‑SNP support.
  • The server must have at least 8 CPU cores, 32 GB RAM, and 512 GB SSD storage.
  • A static IP address and a valid TLS certificate (self‑signed is acceptable for initial setup).
  • A valid Verity licence key, provided by Intellectica AI LLC.
  • systemd (default on most distributions) or the ability to run a long‑lived process.

1. Obtain the Verity Binary

  1. Open a web browser and navigate to verity.io/download (or the URL provided in your licence email).
  2. Enter your licence key exactly as provided. The download will begin automatically.
  3. Save the file verity-<version>.bin to your local machine.

2. Verify Binary Integrity

A SHA‑256 checksum file is available next to the download. Verify the binary before executing it:

sha256sum -c verity-<version>.bin.sha256

Expected output: verity-<version>.bin: OK

3. Install Verity

Copy the binary to a system directory, make it executable, and run the installer:

# Copy binary
sudo cp verity-<version>.bin /usr/local/bin/verity
sudo chmod +x /usr/local/bin/verity

# Run the installer (replaces <licence-key> with your actual key)
sudo verity install --license-key "VERITY-..."

The installer will:

  • Validate the licence key cryptographically.
  • Bind the licence to the server’s hardware.
  • Generate a configuration file at /etc/verity/config.toml.
  • Initialise the Merkle ledger.
  • Print “Verity installed successfully” upon completion.

4. Start the Platform

The recommended way is to use systemd:

# Create a systemd service file
sudo tee /etc/systemd/system/verity.service << 'EOF'
[Unit]
Description=Verity Core Banking Platform
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/verity serve
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable verity
sudo systemctl start verity

Or run it directly in the foreground:

verity serve

5. Access the Mission Control Dashboard

Open a web browser and navigate to:

https://<your-server-address>:8080

The dashboard is served directly by the Verity binary — no separate web server is required.

6. Licence Management

Check the status of your licence at any time:

verity license status

Output example:

Organisation: First Interstate Bank
Expiry:       2027-06-01T00:00:00Z
Hardware match: 100%
Signature:     ✅ valid

If the licence is due to expire, contact Intellectica AI LLC for a renewal key. The new key can be applied without reinstalling.

7. Troubleshooting

“Licence is bound to different hardware”

The server’s hardware fingerprint has changed. This can happen after major hardware upgrades. Contact support for a re‑issued licence.

“System clock appears to have been rolled back”

Ensure NTP is enabled and the system clock is correct. The licence enforcement detects clock manipulation.

“Virtualised/container environment detected”

Verity is running in a VM or container. This is acceptable for evaluation; a warning is printed but the platform will start. Production deployments should use bare‑metal with TEE support.
